## Wave 3 — Experience Agent (Block 2 dashboard service)

**Prompt:**

```
You are the CLIVE Experience Agent. Read DECISIONS.md. v0.11 is in sprint (D-146, D-147).

Build the web dashboard service from scratch at `src/dashboard/`.

**Package structure:**
- `src/dashboard/clive_dashboard/__init__.py` — empty
- `src/dashboard/clive_dashboard/auth.py` — session management
- `src/dashboard/clive_dashboard/api.py` — API handlers
- `src/dashboard/clive_dashboard/push.py` — receive push events from Block 13
- `src/dashboard/clive_dashboard/main.py` — aiohttp server, port 8084
- `src/dashboard/clive_dashboard/static/index.html` — full SPA
- `src/dashboard/pyproject.toml`
- `src/dashboard/Dockerfile`
- `src/dashboard/tests/__init__.py`
- `src/dashboard/tests/test_v11_dashboard_auth.py`
- `src/dashboard/tests/test_v11_dashboard_api.py`

**auth.py requirements:**
- DB-backed web_sessions (clive_state.web_sessions) via asyncpg
- `create_session(user_id)` → 64-char hex token, INSERT into web_sessions, clean expired
- `validate_session(token)` → SELECT + UPDATE last_seen_at; return None if not found
- `invalidate_session(token)` → DELETE
- `require_session(request)` → raises HTTPUnauthorized if no valid cookie
- `handle_login(request)` → secrets.compare_digest, issues HTTP-only cookie
- `handle_logout(request)` → clears cookie
- FAILS CLOSED (no authentication means no access)

**api.py requirements:**
- `_pending_responses: dict[str, dict]` — in-memory store by conversation_id
- `ORCHESTRATOR_URL = os.environ.get("ORCHESTRATOR_URL", "http://orchestrator:8080")`
- `set_pending_response(conversation_id, data)` — called by push.py when Block 13 delivers response
- `handle_query` — emits query.received with source_surface="dashboard" to Block 13 /events
- `handle_poll_response` — GET /api/response; 202 if pending, 200 + response_text if ready
- `handle_history` — proxies to Block 13 /retrieve/conversation-history
- `handle_pending` — proxies to Block 13 /retrieve/pending-actions
- `handle_confirm` — emits action.owner_response decision="confirmed" to Block 13
- `handle_cancel` — emits action.owner_response decision="rejected" to Block 13
- D-006: confirm/cancel NEVER touch pending_actions directly — always route via Block 13

**push.py requirements:**
- `handle_response_push` — called by Block 13 via /push/response; calls set_pending_response
- `handle_confirmation_push`, `handle_alert_push`, `handle_action_outcome_push`, `handle_deletion_result_push`
- Keep last 20 alerts in `_recent_alerts`

**static/index.html:**
- Dark-themed SPA (no external CDN, no external fonts)
- Login screen: password input, error message, submit
- Three-tab app: Query, Pending, History
- Query: conversation display, textarea, 1s polling
- Pending: action list with confirm/cancel, 15s auto-refresh
- History: UUID input, load conversation turns (read-only)
- Vanilla JS only

**Dockerfile:** Same pattern as telegram. Port 8084. CMD python -m clive_dashboard.main

All secrets come from /etc/clive/secrets.env. Never hardcode secrets.
```
